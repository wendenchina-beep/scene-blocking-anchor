# Scene Blocking Anchor

## Core Rule

不是"生成 prompt"，而是**生成一份可交付、可复用的结构化空间档案**（站位记忆表）。用户拿到这张表可以带去任何 AI 生图/生视频工具，保证跨镜空间连贯。

核心方法论：**主视觉锚点 + 相对方位 + 面向关系**。

- **禁用** 现实方位词（南门 / 北墙 / 朝东南）——AI 看不懂
- **禁用** 量化坐标（横向 50% / 纵向 65%）——模型读数字困惑
- **必用** 相对主锚方位词（在主锚正前方 / 面对主锚 / 背对主锚）——模型看图能识别"画面里那个最显眼物体的反方向"

当任务涉及具体分类（角色类型判定、走位描述、跨镜承接）时，`hub_read references/` 下的对应子文档。

## When to Use

任一条件命中 → 建站位记忆表：

- 场景有 ≥3 个角色
- 场景涉及跨场切换（≥2 个场景）
- 含闪回 / 时间跳转
- 含角色走位 / 移动
- ≥2 人对话戏（即使只有 2 人）

单人独白 / 无对手戏 → 跳过。

## Interaction Rule

不要用问卷阻塞用户。用户丢一段场景描述过来 → 直接开始建表。

- 用户提供剧本或场景描述 → 自动识别场景、角色、走位，建表
- 用户没提供剧本，只描述"我有 5 个人在喜堂拜堂" → 用可用信息先出骨架，标注需要补的字段
- 拿不准的字段（角色类型、主锚选择）→ 内部按默认规则处理，输出后在末尾附一句"以下字段建议你复核：..."，不要来回问

## Output Format

Skill 的最终产出是**一份 Markdown 文档**，用户可以直接保存、粘贴到项目、喂给下游任何 AI 工具。文档结构固定：

```markdown
# 站位记忆 — <场景/项目名>

## 场景 1: <场景名>

### 主视觉锚点
- **主锚**: <具体物体名或"<主角>自身位置">
- **主锚在画面的位置**: <画面深处中央 / 中央偏下 / ...>
- **场景光源**: <光源描述>

### 角色站位表
| 角色 | 角色类型 | 相对主锚的位置 | 朝向 | 本场景离场时机 | 走位变化 |
| --- | --- | --- | --- | --- | --- |
| ... | dialogue_actor / on_scene_witness | ... | ... | ... | ... |

### 视线 / 面向关系表
| 角色对 | 默认面向 | 视线（俯/平/仰） | 例外说明 |
| --- | --- | --- | --- |

### 跨 Panel 走位承接
| 角色 | 上一 Panel 末位 | 下一 Panel 起位 | 期间动作 |
| --- | --- | --- | --- |

## 场景 2: <场景名>
（同上模板）

## 跨场景规则
（复用/回归/闪回/道具传递）

## 道具传递链
| 道具 | Panel 1 | Panel 2 | ... |
```

不要输出量化坐标。不要输出现实方位词。不要输出下游 prompt（那是用户自己的事）。

## Main Anchor Selection

每场戏定 1 个主锚，后续所有角色站位/走位/视线都围绕主锚描述。

两种模式，默认优先 A：

| 模式 | 适用 | 例子 |
| --- | --- | --- |
| **A. 场景物体当主锚**（默认，优先选） | 场景里有"最显眼 + 跨镜不动 + 戏剧有意义"的物体 | 灵堂=**灵位** / 病房=**病床** / 法庭=**法官席** / 会议室=**主席台** / 喜堂=**花烛** |
| **B. 主角自身当主锚**（退化兜底） | 场景没固定显眼物 / 场景在动（街道追逐 / 室外行进 / 抽象空间） | 街道追逐=**<主角>位置** / 意识空间=**<主角>蜷缩点** / 山林追杀=**<主角>站位** |

选 A 优先。找不到显眼物体才用 B。

详见 `references/anchor-selection.md`。

## Character Type Classification

区分 dialogue actor vs on-scene witness 是防止"旁观者忽然消失"的关键：

| 类型 | 定义 | 站位表怎么写 |
| --- | --- | --- |
| **dialogue_actor**（对话主体） | 本组镜头里有台词 / 是动作主语 / 是镜头主体 | 完整站位 + 面向 + 走位 |
| **on_scene_witness**（在场旁观者） | 同空间但本组不发声 / 不参与动作。**剧本未明写离场前默认在场** | 完整站位（画面边缘 / 后景 / 虚焦处），本场景全程在场 |

铁律：**剧本没明写"转身走出/离开/退出"→ 默认在场**。同空间 5 个人不能因为本组只有 2 人对话就把其他 3 人从表里删掉。

详见 `references/character-types.md`。

## Position Vocabulary

**必用词**（模型能识别）：

| 维度 | 词汇 |
| --- | --- |
| 画面位置 | 画面左侧 / 画面中央 / 画面右侧 / 画面左下 / 画面右上 |
| 相对主锚（A 模式） | 在主锚正前方 / 左侧 / 右侧 / 身后 / 前方 N 步 |
| 相对主锚（B 模式） | 在主角左侧 1 步 / 右后方 / 围主角形成半圆 |
| 面向关系 | 面对主锚 / 背对主锚 / 侧对主锚 / 面对<角色> / 侧身对<角色> / 背对<角色> |
| 视线 | 平视 / 俯视 / 仰视 / 视线落在<对方>左手上 |

**禁用词**（AI 看不懂或歧义大）：

- ❌ 南门 / 北墙 / 朝东南（现实方位词，AI 看不懂）
- ❌ 横向 50% / 身体侧 90°（量化坐标，模型读数字困惑）
- ❌ 在<对方>旁边 / 靠近<对方>（无方向 / 无距离）
- ❌ 中央偏下 / 附近（模糊）

详见 `references/position-vocabulary.md`。

## Movement Description（走位描述三要素）

每条走位描述必须同时含**朝向 + 距离动词 + 距离刻度**，三要素齐才能锁死画面。

- 「朝主锚**走近 N 步**」（朝向 + 距离动词 + 刻度）
- 「被**朝背离主锚方向拖走 N 步**，跟主锚的距离**拉大**到画面深处」

只写朝向 = 距离歧义（"背对主锚"不代表"离主锚远"，可能后退但仍很近）。

**距离刻度**只用步数 / 臂 / 场景比例，禁用米/厘米/百分比：

| 距离档 | 描述词 |
| --- | --- |
| 极近 | 一臂之距 / 半步 / 贴身 |
| 近 | 1-2 步 |
| 中 | 3-5 步 / 半个场景内 |
| 远 | 半个场景外 / 6+ 步 / 画面深处 |
| 出画 | 走出画面 / 主锚从视野中消失 |

详见 `references/movement-vocabulary.md`。

## Cross-Panel Continuity（跨镜承接铁律）

写下一个镜头前必查上一个镜头末态：

1. **角色位置**：上镜末画面左侧的人 → 下镜必须还在画面左侧（除非剧本明写"走到画面右侧"）
2. **面向关系**：上镜面对面的两人 → 下镜必须还面对面（除非剧本明写"转身 / 背过身 / 走开"）
3. **视线方向**：上镜 A 俯视 B → 下镜必须还俯视（除非剧本明写 A 起身 / B 站起来 改高度差）
4. **走位轨道**：上镜末位 = 下镜起位，期间动作必须可视化

任一违反 = 漂。漂了不是模型的问题，是空间档案没锁住。

详见 `references/cross-panel-continuity.md`。

## Downstream Usage（用户拿到表能干嘛）

Skill 只出档案，不出 prompt。但档案里的字段是可以直接照抄进下游任何工具的：

| 下游工具 | 怎么消费站位表 |
| --- | --- |
| Sora / 可灵 / Runway / Seedance | 每 Panel 写 prompt 前查表，把"角色相对主锚位置 + 面向"照抄进 prompt 自然段 |
| Midjourney / SD / Nano Banana（AI 漫画） | 每格 prompt 里加一句相对方位描述 |
| 分镜师 / 美术 | 直接当"场面调度参考"，画分镜前对齐位置 |
| 编剧 | 反向检查剧本里的空间描述是否有矛盾 |

Skill 不管下游怎么用。如果用户问"帮我写 Sora prompt"，把表里的字段抄进用户已有的 prompt 骨架里，不要自己发明 prompt 模板。

## Final Check

输出前自检：

- [ ] 每场戏都有 1 个主锚
- [ ] 每个角色都标了 dialogue_actor / on_scene_witness
- [ ] on-scene witness 有明确的"离场时机"字段（默认"全场在场"）
- [ ] 所有位置描述用相对方位词，无现实方位词，无量化坐标
- [ ] 有走位的角色，走位描述含"朝向 + 距离动词 + 距离刻度"三要素
- [ ] 每对会视线交流的角色，锁死了默认面向 + 视线高度
- [ ] 有跨 Panel 走位的角色，跨 Panel 承接段写明"末位 = 下起位"
- [ ] 跨场景规则段写明了道具传递链（如有）
