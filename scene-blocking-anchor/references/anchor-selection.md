# 主视觉锚点选择

主锚是本 Skill 的**元概念**，用于站位表本身（让用户一眼看到"这场戏锚物是什么"）。所有角色的位置、走位、视线都围绕主锚描述。

## 关键写作纪律

**主锚是本表的元概念**，用于站位表本身。下游 prompt 写作时（Sora / 可灵 / Seedance），**禁止**保留"主锚"抽象词，**必须**直接展开成具体物体名。

例：
- 站位表写「<主角>在主锚正前方」
- 下游 prompt 写「<主角>跪在灵位正前方」（替换成具体物体名）

模型读"主锚"困惑，读"灵位"才能拍。

## 两种模式

### 模式 A：场景物体当主锚（默认，优先选）

场景里有"最显眼 + 跨镜不动 + 戏剧上有意义"的物体时用 A 模式。

| 场景 | 主锚 |
| --- | --- |
| 灵堂 | 灵位 |
| 病房 | 病床 |
| 法庭 | 法官席 |
| 会议室 | 主席台 |
| 喜堂 | 花烛 |
| 卧室 | 床 |
| 书房 | 书桌 |
| 餐厅 | 餐桌 |
| 神殿 | 神像基座 |
| 城门 | 城门本体 |

**主锚三要素**（缺一不可）：

1. **最显眼**：画面里第一眼能被识别的物体
2. **跨镜不动**：整场戏都在原地，不会因为镜头切换消失或移动
3. **戏剧上有意义**：跟这场戏的核心矛盾/情感相关（灵堂的灵位、喜堂的花烛都承载戏剧张力）

### 模式 B：主角自身当主锚（退化兜底）

场景没固定显眼物 / 场景在动 → 用主角本人当空间基准。

| 场景 | 主锚 |
| --- | --- |
| 街道追逐 | <主角>位置 |
| 意识空间 / 抽象空间 | <主角>蜷缩点 |
| 山林追杀 | <主角>站位 |
| 荒野奔跑 | <主角>移动轨迹 |
| 梦境 | <主角>意识锚点 |

B 模式下所有其他角色的位置都用「在主角左侧 1 步」「围主角形成半圆」等表达。

## 选择决策树

```
场景是室内 / 有固定场景 ？
├── 是 → 场景里有一个"跨镜不动 + 显眼 + 有戏剧意义"的物体 ？
│         ├── 有 → 用 A 模式，选那个物体
│         └── 没有 → 用 B 模式（主角当锚）
└── 否（追逐 / 移动 / 抽象）→ 用 B 模式
```

## 常见错误

- ❌ 用"南门 / 北墙 / 朝东南"当锚 → 现实方位词，AI 看不懂
- ❌ 一场戏定 2 个主锚 → 空间参考系混乱，跨镜必漂
- ❌ 跨场景不换主锚 → 场景 A 的主锚拿到场景 B 用，模型无法理解
- ❌ 选一个"戏里不重要的小物体"当主锚（比如把"茶杯"当会议室主锚）→ 镜头切换时茶杯可能被裁掉，锚失效

## 主锚在画面的位置

选好主锚后，还要标注**主锚在画面里通常出现在哪**，方便下游写 prompt 时构图对齐：

| 位置描述 | 适用 |
| --- | --- |
| 画面深处中央 | 大部分场景物体锚（灵位 / 主席台 / 神像基座） |
| 画面中央偏下 | 主角锚（B 模式） / 桌面类锚（餐桌 / 书桌） |
| 画面右侧 / 左侧 | 侧位锚（法官席 / 主位靠边） |
| 画面前景 | 极少数近景锚（用于近景戏） |

写入站位表「主锚在画面的位置」字段。
