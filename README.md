# 🎬 Scene Blocking Anchor

> 把多角色场景变成结构化空间档案 —— 一个专注于 AI 生视频/生图跨镜空间连贯性的 Skill

---

## 这是什么

**Scene Blocking Anchor** 是一个为 AI 编程助手（Codex / Claude Code / MiniMax Code 等）设计的 Skill。

它解决的核心痛点是：

> 用 Sora / 可灵 / Runway / Seedance / Midjourney 做多角色场景时，角色老是**位置乱跳、面向跳变、旁观者忽然消失**，怎么办？

根因是每次生成都是独立调用，模型没有跨镜记忆。这个 Skill 让你先建一份**站位记忆表**，把空间信息锁死后再送给下游任何 AI 工具，跨镜连贯性直接翻 10 倍。

## 工作原理

不是"帮你写 prompt"，而是**帮你建立可交付、可复用的结构化空间档案**：

```
输入剧本 → 识别在场角色 → 选主视觉锚点 → 分角色类型
       → 用相对方位词写站位 → 锁面向 / 视线关系
       → 写跨 Panel 走位承接 → 输出可直接消费的 Markdown 空间档案
```

**核心方法论：主视觉锚点 + 相对方位 + 面向关系**

- **禁用**现实方位词（南门 / 北墙 / 朝东南）—— AI 看不懂东西南北
- **禁用**量化坐标（横向 50% / 身体侧 90°）—— 模型读数字困惑
- **必用**相对主锚方位词（在主锚正前方 / 面对主锚 / 背对主锚）—— 模型看图能识别"画面里那个最显眼物体的反方向"

## 能做什么

### 🎯 主视觉锚点（Main Anchor）
每场戏选 1 个空间基准点，所有角色位置都围绕它描述。两种模式：
- **A 模式**：场景物体当主锚（灵堂=灵位 / 病房=病床 / 喜堂=花烛）
- **B 模式**：主角自身当主锚（追逐 / 意识空间 / 抽象场景）

### 👥 角色类型分类
区分 dialogue actor（对话主体）vs on-scene witness（在场旁观者），防止"旁观者忽然消失"这类跨镜穿帮。

### 📐 相对方位词典
一套完整的"AI 能读懂"的方位词汇：
- 画面位置（画面左侧 / 中央 / 右侧 / 左下）
- 相对主锚（正前方 / 左侧 / 右侧 / 身后 / 左前方 N 步）
- 面向关系（面对 / 侧对 / 背对）
- 视线角度（平视 / 俯视 / 仰视 / 视线落在…上）

### 🚶 走位描述三要素
每条走位必须同时含：
- **朝向**（面朝哪边）
- **距离动词**（推远 / 拉近 / 收窄 / 远离 / 靠近）
- **距离刻度**（N 步 / 一臂之距 / 半个场景外 / 出画）

三要素齐才能锁死画面，只写"朝南走"AI 拍不出来。

### 🔗 跨镜承接铁律
锁死跨 Panel 的位置、面向、视线、走位承接，防止：
- 上镜画面左的人 → 下镜跳到画面右
- 上镜面对面 → 下镜莫名背对
- 上镜俯视 → 下镜平视（没剧本依据）
- 上镜末位 → 下镜起位不一致 = 瞬移

### 📦 道具传递链
跨场景 / 跨 Panel 的道具位置、状态变化（戴 / 摘 / 完整 / 碎裂）也锁死。

## 安装

将 `scene-blocking-anchor` 文件夹放到你的 Skill 目录：

```bash
# Codex
~/.codex/skills/scene-blocking-anchor

# MiniMax Code (Mavis)
~/.mavis/skills/scene-blocking-anchor
```

目录结构：

```
scene-blocking-anchor/
├── SKILL.md                                # 技能主文件（核心规则 + 触发判定）
└── references/
    ├── anchor-selection.md                 # 主锚选择（A/B 模式决策树）
    ├── character-types.md                  # 角色类型（dialogue vs witness）
    ├── position-vocabulary.md              # 相对方位词典（必用 vs 禁用）

    └── full-example.md                     # 完整示例（喜堂拜堂 5 Panel）
```

## 使用方法

### 4 步线性流程

```
Step 1: 准备输入          → 你手上要有什么
   ↓
Step 2: 触发 Skill        → 一句话唤起
   ↓
Step 3: Skill 输出档案    → 拿到一份结构化 Markdown
   ↓
Step 4: 送去下游 AI 工具  → 把档案字段抄进 prompt
```

**关键理解**：这个 Skill 不生成视频，也不生成 prompt——它只生成**中间产物**（结构化空间档案）。这份档案是给你后面写 Sora / 可灵 / Runway / Seedance / Midjourney prompt 时**照抄**用的。

---

### Step 1：准备输入

在触发 Skill 之前，你需要有以下三样东西之一（有一样就够）：

| 输入形式 | 举例 |
| --- | --- |
| **A. 完整场景剧本** | "第 5 集喜堂拜堂，5 人在场（新娘/新郎/婆母/丫鬟/婆子），婆母揭穿身份 [粘贴对话]" |
| **B. 镜头列表** | "5 个 Panel：P1 拜堂 / P2 婆母上前 / P3 揭盖头 / P4 拖走新娘 / P5 新娘回望" |
| **C. 场景描述** | "餐厅打斗，4 个角色围着一张桌子，主角护住一个孩子，两个反派从左右夹击" |

**你不需要**准备：
- ❌ 主锚是什么（Skill 自己选）
- ❌ 每个角色是 dialogue actor 还是 witness（Skill 自己判定）
- ❌ 相对方位怎么写（Skill 自己生成）

---

### Step 2：触发 Skill

直接一句话，把输入丢过去。示例：

```
$scene-blocking-anchor 帮我为这段场景建站位表：
[粘贴你的剧本 / 镜头列表 / 场景描述]
```

或者自然语言触发（不加 `$` 也行）：

```
用 scene-blocking-anchor 处理这段多人场景 [粘贴内容]
```

---

### Step 3：Skill 输出档案

Skill 会自动完成 7 件事，输出一份 Markdown 文档：

| # | Skill 内部做的事 | 你看到的输出 |
| --- | --- | --- |
| 1 | 识别在场角色 + 场景 | 「主视觉锚点」段（主锚 + 位置 + 光源） |
| 2 | 选主锚（A/B 模式决策） | 主锚字段（如"花烛"或"<主角>自身位置"） |
| 3 | 分角色类型 | 「角色站位表」的"角色类型"列 |
| 4 | 用相对方位词写站位 | 「角色站位表」的"相对主锚位置"列 |
| 5 | 锁面向/视线关系 | 「视线/面向关系表」 |
| 6 | 写跨 Panel 走位承接 | 「跨 Panel 走位承接表」 |
| 7 | 输出道具传递链（如有） | 「道具传递链」段 |

**输出结构一览**：

```markdown
# 站位记忆 — <场景名>

## 场景：<场景名>

### 主视觉锚点
- 主锚：花烛
- 主锚在画面的位置：画面深处中央
- 场景光源：...

### 角色站位表
| 角色 | 角色类型 | 相对主锚的位置 | 朝向 | 离场时机 | 走位 |
| 新娘 | dialogue_actor | 在主锚正前方一臂之距跪位 | 面对主锚 | 全场在场 | Panel 4 被拖走 |
...

### 视线/面向关系表
| 角色对 | 默认面向 | 视线 | 例外说明 |
...

### 跨 Panel 走位承接
| 角色 | Panel N 末位 | Panel N+1 起位 | 期间动作 |
...

### 绝对方位锁
| 角色 | 永远在画面的 | 永远位于主锚的 |
...

## 道具传递链
| 道具 | Panel 1 | Panel 2 | ... |
```

完整示例见 [references/full-example.md](scene-blocking-anchor/references/full-example.md)。

---

### Step 4：把站位信息注入到下游 prompt

拿到档案后，**让 AI 结合这份档案，把每一段镜头 prompt 里的站位信息补进去 / 修正掉**。

用法很简单——不管你手上是完整的 prompt 骨架，还是随手写的一句话 prompt，都是同一个动作：

> 「这是我的第 X 组镜头 prompt：[粘贴]。这是这场戏的站位记忆表：[粘贴]。请按站位表把这段 prompt 里的角色位置、面向、走位固定好。」

AI 会自动做的事情：

- **主锚具体名替换**：把站位表里的「主锚」抽象词换成具体物体（「花烛」/「灵位」/「病床」）
- **相对方位词注入**：把「新娘在主锚正前方一臂之距跪位」照抄进 prompt
- **旁观者补齐**：本组只有对话主体的 prompt，AI 会自动加一句「画面左下虚焦处，丫鬟站立观望」
- **面向关系锁定**：「新娘正面对婆母，婆母俯视新娘」逐字进 prompt
- **跨镜承接对齐**：Panel N+1 起位 = Panel N 末位，走位动作按站位表照抄

这样每组镜头的 prompt 都是**同一份空间档案的一致投影**，跨镜位置不漂、面向不跳变、旁观者不消失。

档案本身也是标准 Markdown，如果你要直接给美术/分镜师看 / 存进 Notion、飞书文档，也可以直接用。

## 核心原则

| 原则 | 说明 |
| --- | --- |
| 🎯 **主锚统一** | 每场戏 1 个主锚，所有角色位置围绕它描述 |
| 🔤 **禁现实方位词** | 南门 / 北墙 / 朝东南 → AI 看不懂 |
| 🔢 **禁量化坐标** | 横向 50% / 身体侧 90° → 模型读数字困惑 |
| 📏 **走位三要素齐** | 朝向 + 距离动词 + 距离刻度 缺一不可 |
| 👀 **旁观者不消失** | 剧本未明写离场 → 默认在场 |
| 🔒 **跨镜承接锁死** | 末位 = 下起位，面向不可自由改 |

## 支持的场景类型

| 类型 | 触发条件 | 是否出表 |
| --- | --- | --- |
| 多人对话 | ≥3 个角色 | ✅ 必出 |
| 跨场切换 | ≥2 个场景切换 | ✅ 必出 |
| 闪回 | 含时间跳转 | ✅ 必出 |
| 走位戏 | 含角色移动 | ✅ 必出 |
| 2 人对话 | ≥2 人对话戏 | ✅ 必出 |
| 单人独白 | 全场无对手戏 | ❌ 跳过 |

## 输出格式

Skill 的最终产出是**一份 Markdown 文档**，结构固定：

1. **主视觉锚点段**（主锚 / 位置 / 光源）
2. **角色站位表**（角色 / 类型 / 相对主锚位置 / 朝向 / 离场时机 / 走位）
3. **视线/面向关系表**（每对角色的默认面向 + 视线高度）
4. **跨 Panel 走位承接表**（末位 → 下起位 + 期间动作）
5. **绝对方位锁**（每角色永远在画面 X 侧 + 永远位于主锚的 Y 方位）
6. **跨场景规则**（场景 A → B → A 回归 / 闪回切回 / 主锚切换）
7. **道具传递链**（跨 Panel 的道具位置和状态变化）

## Skill 的边界（我不做什么）

**不做的事情**：
- ❌ 不做**分镜拆分**（把 30 秒剧本拆成 8 个 Panel 是短剧/广告/漫画各自的领域，跟本 Skill 无关）
- ❌ 不做**下游 prompt 编写**（Sora / 可灵 / Seedance 各有各的模板，用户自己套）
- ❌ 不做**剧本创作**（本 Skill 吃剧本，不写剧本）

**做的事情**：
- ✅ 拿到"角色 + 场景 + Panel 列表" → 输出结构化空间档案
- ✅ 输入形式可以是完整剧本 / 场景描述 / 镜头列表

如果你还不知道怎么把剧本拆成镜头，那是**上游的问题**，本 Skill 帮不了你（可以参考各类分镜工具/方法论）。
