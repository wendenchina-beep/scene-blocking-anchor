# 角色类型分类

**核心问题**：多角色场景（≥3 人）里，很多"同空间但本组镜头不发声"的角色容易被 agent 从 prompt 里剥离 → 跨组镜头空间断裂 → 用户感觉"对话双方在不同房间"。

**根因**：把"对话参与者"等同于"画面在场角色"。

**解决**：站位表里每个角色必须标类型，两类：

## 两种角色类型

### dialogue_actor（对话主体）

**定义**：本组镜头里有台词 / 是动作主语 / 是镜头主体

**站位表怎么写**：
- 完整站位（相对主锚位置 + 距离）
- 完整朝向（面对/背对/侧对 主锚 或 其他角色）
- 完整走位（有走位时，含"朝向 + 距离动词 + 距离刻度"三要素）
- 离场时机：默认"全场在场"，除非剧本明写走出

**下游 prompt 处理**：
- Prompt 里 **主要描述**（眼神 / 表情 / 台词 / 动作占 70%+ 篇幅）
- 相对主锚位置 verbatim 复制自站位表

### on_scene_witness（在场旁观者）

**定义**：同空间但本组镜头不说话 / 不参与动作。**剧本未明写离场前默认在场**

**站位表怎么写**：
- 完整站位（画面边缘 / 后景 / 虚焦处的具体位置）
- 简单朝向（观望 / 侧身 / 低头静立）
- 通常无走位
- 离场时机：默认"全场在场"

**下游 prompt 处理**：
- Prompt 里 **至少 1 句** 背景描述（画面左下角 / 右下角 / 后景虚焦处）
- 位置**跨组必须一致**（如站位表定的"C 在主锚左后方 3 步" → 每个 Panel 都是"画面左下角虚焦处"）

## 离场时机判定

写下一个 Panel 前必查：

1. **剧本明写"<角色> 转身走出 / 离开 / 退出"** → 该 Panel 后从"在场名单"移除，后续 Panel 不再挂
2. **剧本明写场景切换**（如"切到走廊" / "切到次日"） → 整个新场景重新定在场名单
3. **剧本未明写离场** → **默认在场**，即使本组镜头不发声也必须挂表 + 位置描述

## 举例

**场景**：喜堂拜堂，5 人在场（新娘、新郎、婆母、丫鬟、婆子），本 Panel 是婆母揭穿新娘身份

| 角色 | 角色类型 | 依据 |
| --- | --- | --- |
| 婆母 | dialogue_actor | 揭穿台词是她说的 |
| 新娘 | dialogue_actor | 被揭穿的对象，镜头主体 |
| 新郎 | on_scene_witness | 拜堂在场，本组不发声 |
| 丫鬟 | on_scene_witness | 陪嫁在场，本组不发声 |
| 婆子 | on_scene_witness | 侍立在场，本组不发声 |

**错误做法**：把新郎、丫鬟、婆子从站位表里删掉，下 Panel 用户看到新郎突然出现 → 感觉"角色瞬移"。

**正确做法**：全部 5 人都进站位表，标类型，各 Panel 都保持位置一致，直到明写离场为止。

## 禁止形态

- ❌ 把"对话参与者"等同于"画面在场角色" → 同空间其他角色被剥离 → 跨组镜头空间断裂
- ❌ 「本组只有 A + B 对话，所以只写这 2 人」（忽略同空间 C / D 的在场）
- ❌ 角色一会儿在画面，一会儿凭空消失，中间没显式离场动作 → 用户感觉"角色在两个不同空间穿梭"
